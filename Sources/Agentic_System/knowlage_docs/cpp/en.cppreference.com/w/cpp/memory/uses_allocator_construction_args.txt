[ edit ] Notes The overloads (2-9) provide allocator propagation into std::pair , which supports neither leading-allocator nor trailing-allocator calling conventions (unlike, e.g. std::tuple , which uses leading-allocator convention). When used in uses-allocator construction, the conversion function of pair-constructor converts the provided argument to std::pair at first, and then constructs the result from that std::pair by uses-allocator construction.
